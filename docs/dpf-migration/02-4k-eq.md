## Source inventory

`plugins/4k-eq/` (flat layout, no Source/ dir):
- `FourKEQ.h/cpp` — processor: EQ bands + console saturation. JUCE deps:
  `juce::dsp::IIR` (the whole filter section), APVTS params (~12 real params).
- `ConsoleSaturation.h` — saturation stage (check: mostly plain math already).
- `EQCurveDisplay.h/cpp` — response-curve UI component.
- `FourKLookAndFeel.h/cpp`, `PluginEditor.*` — JUCE UI (replaced by ImGui).
- `FourKEQPresets.h` — preset values (port into the DPF preset table format).
- JUCE identity: check `CMakeLists.txt` for `PLUGIN_CODE`/`LV2URI`; the DPF
  build must use different IDs.

## DSP port plan

1. New `plugins/4k-eq/core/FourKEQDSP.hpp/cpp` (framework-free):
   - Each `juce::dsp::IIR` band → shared RBJ `Biquad` (peak/shelf per band).
     Recompute coefficients at block rate when a band param changed
     (tape-echo's `lastBass/lastTreble` dirty-check pattern).
   - **Mandatory project rule: IIR EQs oversample** ("no EQ cramping ever" —
     pre-warping alone is insufficient). Run the whole filter+saturation
     chain at 2x minimum via shared halfbands; measure response at 15–20 kHz
     vs the JUCE build (which already oversamples) to confirm no cramping
     regression. Compensate the fixed group delay via `setLatencySamples`
     equivalent — DPF: `DISTRHO_PLUGIN_WANT_LATENCY 1` + `setLatency()`.
   - Console saturation: port as-is if plain math; if it saturates full-band
     input, it lives inside the oversampled section anyway.
   - Bypass param with host designation, bit-exact passthrough.
   - Output peak atomic for a VU/peak LED if the JUCE UI had metering.
2. Match the JUCE parameter list exactly (names, ranges, defaults, tapers) —
   read them from `FourKEQ.cpp`'s layout function. Same values must produce
   the same curve.

## UI plan

- Console-strip look: band knob columns (LF/LMF/HMF/HF + filters), bypass
  buttons per band if the original has them.
- EQ curve display: compute the composite biquad magnitude response on the
  UI thread (evaluate |H(e^{jw})| from the same coefficient math — do NOT
  probe the audio path) and draw as an ImDrawList polyline, log-frequency
  x-axis, ±18 dB y. Redraw when any param changes.
- Shared widgets + crisp font from `shared-dpf`.
